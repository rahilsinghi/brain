# Maison Onboarding — Complete Context (April 15, 2026)

## What Maison Is
Maison is an AI-powered commercial platform for travel and hospitality. It deploys an AI agent across hotel websites, mobile apps, SMS, WhatsApp, email, and internal tools. Guests interact naturally — asking questions, checking availability, making requests — and the AI handles it. Hotel operators manage via a purpose-built console.

## Repo Access & Setup
- **Monorepo:** `maison-agent` on GitHub (github.com/Maison-Labs/maison-agent)
- **Clone and bootstrap:** `git clone` → `make bootstrap` → `make dev`
- **Tech stack:** TypeScript-first, pnpm workspaces + Turborepo, AWS CDK infrastructure
- **Local URL:** https://local.maison-labs.com
- **.env:** Generated by `make bootstrap`. Requires Firebase service account key file (stored outside repo). See #claude-onramp in Slack for full template.
- **Branch convention:** `service/feature-name` (e.g., `console/kb-analyzer`)
- **Key domains:** maison.cx (public site), maison-labs.com (platform)

## Architecture Overview
- **API:** Node.js on ECS Fargate. REST + A2A (JSON-RPC) + MCP interfaces.
- **Database:** PostgreSQL on Neon (primary), DynamoDB (sessions/tasks), Pinecone (vector search), S3 (files)
- **Auth:** Firebase Auth for operators, HMAC API keys for programmatic access
- **Console:** A2UI schema-driven rendering (JSON schemas → HTML). Thin shell app.
- **Real-time:** AppSync Events (WebSocket) for chat message delivery
- **Observability:** OpenTelemetry → Grafana Tempo (traces) + Loki (logs) + Langfuse (LLM tracing)
- **Email:** SES
- **Background:** SQS + Lambda
