# Maison Agents — Commercial Packaging & Terminology
**Source:** Notion page "Maison Agents (Commercial Packaging)"
**Retrieved:** April 15, 2026

## Three-Layer Technical Model
| Layer | Term | Example |
|---|---|---|
| Bottom | Tool | PMS lookup, WhatsApp send, CRM update |
| Mid | Skill | "Handle upsell offer", "Answer FAQ", "Generate review request" |
| Top | Agent | Support Agent, Sales Agent, Concierge Agent |

**Tools** — atomic, discrete functions. Stateless, low-level, reusable.
**Skills** — packaged business-process capabilities orchestrating several tools.
**Agents** — role-based, autonomous AI workers with persona, goal, skills, and judgment.

## Commercial Vocabulary (Operator-Facing)
- **Pre-built Agents** — eleven core agents, fully managed by Maison
- **Custom Agents** — spun up by operator, no code required
- **Mini-agents** — single trigger + action (e.g., "text guests parking info 2 hours before arrival")

**Rule of thumb:**
- Internally (engineering, Linear, docs): Tool → Skill → Agent
- Externally (sales, marketing, onboarding): Pre-built Agent → Custom Agent → Mini-agent

## The Maison Agent Stack Vision
Maison = modular platform of AI agents to run hotel commercial operations end to end. Distribution, sales, marketing, revenue, intelligence, guest communications — each owned by a dedicated agent.

Entry point: forever-free FAQ webchat.
Destination: full commercial platform giving independent hotels the firepower of a large chain without legacy systems or OTA dependency.
