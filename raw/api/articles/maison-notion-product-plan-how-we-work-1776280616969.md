# Maison Product Plan — How We Work
**Source:** Notion page "Product Plan"
**Retrieved:** April 15, 2026

## Planning Cycle
1. **Capture** — Commercial team logs requests in Product Request Tracker (Notion). One row per request.
2. **Groom** — Fredrik reviews tracker + Feature List. High-frequency/priority → shaped into features or epics.
3. **Scope** — Features spec'd (PRD or lightweight requirements), linked to Linear project/issue. Size = scope proxy: XL/L = multi-sprint epic, M/S = standalone feature.
4. **Build** — Work prioritized in Linear in ongoing Cycle (no sprint planning).
