# Maison QA & Test Automation

This article outlines Maison's Quality Assurance and Test Automation system, its current status, and future vision.

## System Overview
Maison utilizes an **agentic, AI-driven QA system** built upon [[Playwright]]. Tests are generated by [[Claude]] using natural language descriptions and executed from a centralized web dashboard. A key feature is that **no code is needed to create or run tests**, making it accessible to a broader audience. All test results, including screenshots and screen recordings, are automatically stored for review.

## Quality Dashboard URLs
The [[Quality Dashboard]] is the central hub for managing and viewing test results. Access credentials (`QUALITY_ADMIN_EMAIL` / `QUALITY_ADMIN_PASSWORD`) are sourced from the `.env` file.

*   **Local:** `local.maison-labs.com`
*   **Test:** `quality.test.maison-labs.com`
*   **Staging:** `quality.stg.maison-labs.com`
*   **Production:** `quality.maison-labs.com`

## Creating Tests
Tests are created by providing plain language descriptions to [[Claude]]. Specificity is crucial for effective test generation.

**Example Prompt:**
> "Create a new quality UI test. Steps: open the login page at the Maison Labs login URL. Verify the Maison wordmark is visible. Verify the wordmark changes between light mode and dark mode."

**Tips for Test Creation:**
*   Be specific about each step.
*   Include credentials if required for the test scenario.
*   Keep each test focused on a single objective or flow.

Test suites are organized by type within `apps/quality/src/suites/`, including categories such as smoke, functional, visual, API, and data tests.

## Priority Test Areas (April 13)
Key areas identified for comprehensive testing:
*   **[[Client list page]]**: The first live feature in the console.
*   **[[Login page]]**: Verification of light and dark mode functionality.
*   **[[Chat widget]]**: Loading on demo sites and send/receive message functionality.
*   **[[Guest-facing flows]]**: End-to-end testing of the guest experience.

## Key Dates
*   **Goal:** Achieve a comprehensive test suite, ready by approximately April 22nd, for the staging to production deployment.
*   **Current Status:** As of April 15, 2026, there are 194 test cases, with a recognized need for significantly more to meet coverage goals.

## AI-Native Testing Strategy (Future)
The future vision for Maison's QA system involves advanced AI capabilities to enhance efficiency and coverage:
1.  **Spec-to-test generation:** Automatically generating tests directly from [[Linear ticket ACs]] (Acceptance Criteria).
2.  **[[Self-healing tests]]**: Tests that can automatically repair themselves (e.g., updating selectors or endpoints) when minor UI changes occur.
3.  **Intelligent test selection**: AI analyzes code diffs and selects the minimum necessary set of tests to run, optimizing testing cycles.
4.  **Autonomous exploratory testing**: AI navigates the application without pre-defined scripts, identifying potential issues through [[exploratory testing]].
