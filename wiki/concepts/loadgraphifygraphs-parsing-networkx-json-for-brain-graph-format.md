# loadGraphifyGraphs: Parsing NetworkX JSON for Brain Graph Format

The `loadGraphifyGraphs` function is responsible for parsing NetworkX JSON output from `graphify` into a standardized Brain Graph format. It processes graph data files from a specified output directory, converting them into `GraphNode[]` and `GraphLink[]` structures, while ensuring per-repository isolation and handling various graph data specifics.

## Details

The `loadGraphifyGraphs` function, part of the `rahilsinghi/brain` repository, is designed to integrate graph data generated by the `graphify` tool into the Brain's knowledge graph. It operates by:

*   **Scanning `graphifyOutputDir`**: It searches for `_graph.json` files within the designated `graphify` output directory.
*   **NetworkX to Brain Format Mapping**: It takes `NetworkX`'s `node_link_data` format and transforms it into the Brain's internal `GraphNode[]` and `GraphLink[]` array structures.
*   **URI Generation**: Each node is assigned a unique URI using the `code://` prefix, enabling consistent referencing within the Brain's ecosystem.
*   **Layer Assignment**: All imported nodes are tagged with `layer=code`, indicating their origin from codebase analysis.
*   **Per-Repository Isolation**: The system maintains isolation for graphs generated from different repositories.
*   **Symbol-Level Fragment Support**: It supports fragment identifiers (e.g., `#functionName`) to link directly to specific symbols within code files.
*   **Error Handling**: Includes mechanisms to skip malformed JSON files and guard against missing directories, ensuring robustness.

This process is crucial for populating the Brain's graph with structured information derived from code repositories, facilitating knowledge discovery and navigation.
