# MarkPush: iOS 26 Physical Device Compatibility Fixes

This entry details a series of fixes implemented in the MarkPush repository (SHA `55aa4b6`) to ensure compatibility with iOS 26 on physical devices. Key changes address UI rendering issues, font warnings, orientation, and Supabase client configuration, improving the app's stability and appearance.

## Details

This commit (`55aa4b6`) introduces several critical adjustments to the `MarkPush` project to ensure proper functionality and a consistent user experience on physical iOS 26 devices. The changes span UI, font handling, orientation, and development setup:

*   **UI and Navigation Enhancements**:
    *   **NavigationStack Toolbar Replacement**: The default `NavigationStack` toolbar was replaced with a custom header. This change was necessary to eliminate the intrusive "..." button generated by the iOS 26 Liquid Glass system on the `FeedView` when running on physical devices, which interfered with the app's UI.
    *   **Sidebar Toggle Suppression**: The `.tabViewStyle(.tabBarOnly)` modifier was added to prevent an unwanted sidebar toggle from appearing, ensuring a cleaner and more controlled tab bar experience.

*   **Font and Layout Adjustments**:
    *   **Font Weight Warning Fix**: Resolved a warning related to font weights by directly swapping Lora font faces. This approach avoids using `.fontWeight()` on custom fonts, which could lead to unpredictable rendering or warnings.
    *   **Portrait-Only Orientation Enforcement**: The `UIRequiresFullScreen` setting was added, which enforces a portrait-only orientation for the application, optimizing its layout and usability on physical devices.

*   **Backend and Development Configuration**:
    *   **Supabase Client Configuration**: The `emitLocalSessionAsInitialSession` property was set on the Supabase client, which is a crucial configuration for managing user sessions effectively.
    *   **PairingView Dismissal Fix**: The dismissal mechanism for the `PairingView` was rectified by utilizing the `@Environment(\.dismiss)` property, ensuring the view can be properly closed.
    *   **Device Signing**: `DEVELOPMENT_TEAM` was added to the project settings, a necessary step for codesigning the application and deploying it to physical devices for testing or distribution.
