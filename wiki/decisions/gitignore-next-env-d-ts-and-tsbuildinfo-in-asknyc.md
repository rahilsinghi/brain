# Gitignore next-env.d.ts and .tsbuildinfo in askNYC

This commit updates the `.gitignore` file in the [[askNYC]] project to exclude `next-env.d.ts` and `.tsbuildinfo` files. This prevents build-generated TypeScript environment declarations and incremental build information from being committed to the repository.

## Key Concepts

.gitignore,next-env.d.ts,.tsbuildinfo,Build Artifacts,Version Control

## Details

This commit (SHA: `0b0b110`) by Rahil Singhi on 2026-03-28 updated the `.gitignore` file within the `rahilsinghi/askNYC` repository. The primary objective was to add `next-env.d.ts` and `.tsbuildinfo` to the ignored files list.

- `next-env.d.ts`: This file is automatically generated by [[Next.js]] applications to provide TypeScript environment declarations, ensuring proper type inference for Next.js-specific features. As a generated file, it should not be committed to version control to avoid conflicts and keep the repository clean.

- `.tsbuildinfo`: Generated by the [[TypeScript]] compiler when `incremental` builds are enabled in `tsconfig.json`, this file stores information about previous compilations to speed up subsequent builds. It is a temporary build artifact and is best excluded from source control.

Ignoring these files contributes to a cleaner repository, prevents unnecessary merge conflicts, and allows individual development environments to generate these artifacts as needed.
