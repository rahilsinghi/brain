# Anti-Hallucination Constraints in Resume Generation

A commit to the Career-Datacenter repository introduced explicit anti-hallucination rules into the system prompt governing AI-based resume generation. These changes enforce strict fidelity to knowledge base data, preventing local LLMs from fabricating or altering institutions, dates, locations, and experiences. The fix also corrected a vague instruction that caused a fabricated company reference to appear in generated resumes.

## Key Concepts

- **Anti-Hallucination Rules**: Explicit constraints added to system prompts to prevent AI models from inventing or modifying factual data
- **System Prompt Engineering**: Crafting precise instructions to guide LLM behavior and reduce undesirable outputs
- **Knowledge Base Fidelity**: Requirement that generated content must exactly match source data without paraphrasing or substitution
- **Local LLM Limitations**: Tendency of locally-run language models to fabricate details when instructions are ambiguous
- **Resume Generation AI**: Automated system for producing resumes from structured personal/professional data

## Details

## Overview

This commit (SHA: `62fecf8`) was made on **2026-02-13** by **Rahil Singhi** to the `rahilsinghi/Career-Datacenter` repository. It addresses a recurring problem where AI-generated resumes contained hallucinated or incorrect information.

## Changes Made

### Anti-Hallucination Rules Added to System Prompt
The following explicit constraints were introduced:
- The model **must use EXACT data** from the knowledge base with no modifications
- Institution names, dates, locations, and GPAs **cannot be altered**
- The model **cannot invent experiences** or misplace projects into the experience section
- Dates must be used **exactly as provided** (e.g., `May 2025` must not become `June 2023`)

### Corrected Ambiguous Instruction
A vague instruction was identified as the root cause of a fabricated company name appearing in outputs:

| Before | After |
|--------|-------|
| `mention Employee #3 at Antler-backed startup` | `mention Employee #3 at Kismet` |

By replacing the generic descriptor with the actual company name **Kismet**, the model no longer generates a placeholder such as "Antler-backed Startup."

## Observed Hallucination Examples Fixed
- **Wrong institution name**: `Sri Sathya Sai Institute` generated instead of `SSN College`
- **Wrong location**: `Chittoor` generated instead of `Chennai`
- **Fabricated company**: `Antler-backed Startup` generated instead of `Kismet`

## Impact
These changes significantly reduce hallucination rates in resumes generated by local LLMs, improving the accuracy and trustworthiness of the Career-Datacenter system.
