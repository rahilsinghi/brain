# Gitignore Update for TypeScript Build Artifacts in askNYC

This commit updated the `.gitignore` file in the `askNYC` repository to exclude `next-env.d.ts` and `tsbuildinfo` files. This standard practice prevents temporary TypeScript and Next.js build artifacts from being tracked by Git, ensuring a cleaner repository.

## Key Concepts

Gitignore,TypeScript Build Artifacts,next-env.d.ts,tsbuildinfo,Version Control

## Details

The `0b0b110` commit in the `rahilsinghi/askNYC` repository introduced changes to the project's `.gitignore` file. Specifically, it added two entries:

*   `next-env.d.ts`: This file is automatically generated by Next.js projects when using TypeScript. It contains important type definitions for the Next.js environment, but it is derived from the project's source code and configuration, making it a temporary build artifact that does not need to be committed to version control.
*   `tsbuildinfo`: This file is generated by TypeScript when using incremental compilation (`--incremental` flag). It stores information about the compilation process to speed up subsequent builds by only recompiling changed files. Like `next-env.d.ts`, it's a temporary file that should not be tracked by Git.

Excluding these files from the repository prevents unnecessary diffs, merge conflicts, and bloat in the Git history, ensuring that only essential source code and configuration are committed.
