# Add Job Search Configuration Templates to Career-Datacenter

This commit introduces several YAML configuration templates to the `Career-Datacenter` project, designed to streamline job tailoring and interview preparation. These templates cover job preferences, role-specific keyword mappings, ATS optimization, and behavioral interview story structures.

## Key Concepts

- `preferences.yaml`: Defines job search constraints, company preferences, and compensation expectations.,- `role_families.yaml`: Provides keyword mappings tailored for various technical roles like backend, data, ML, quant, fullstack, and infrastructure.,- `keyword_library.yaml`: Maps technical and soft skills for Applicant Tracking System (ATS) optimization.,- `stories_star.yaml`: Offers a templated structure for behavioral interview stories using the STAR method.

## Details

As part of the `rahilsinghi/Career-Datacenter` project, this commit (bfab505) from 2026-02-10 by Rahil Singhi introduced 295 lines across 4 new configuration files, with no deletions. These files serve as foundational templates for enhancing the automated job application and interview preparation processes within the system.

The added templates are:
*   **`preferences.yaml`**: This file allows users to systematically define their job search parameters, including desired compensation, specific company preferences, and other constraints. This ensures that the job search is aligned with personal criteria.
*   **`role_families.yaml`**: To facilitate targeted applications, this template maps keywords relevant to different technical role families (e.g., backend, data science, machine learning, quantitative finance, fullstack development, and infrastructure engineering). This mapping is crucial for tailoring resumes and applications effectively.
*   **`keyword_library.yaml`**: Focused on Applicant Tracking System (ATS) optimization, this file provides a comprehensive mapping of both technical and soft skills. By leveraging this library, the system can ensure that generated resumes and cover letters are keyword-rich and more likely to pass initial ATS screenings.
*   **`stories_star.yaml`**: This template is designed to help users structure their behavioral interview responses using the STAR method (Situation, Task, Action, Result). It provides a standardized framework for preparing compelling and effective stories for common interview questions.

Collectively, these templates enable more systematic and efficient job tailoring and interview preparation, significantly improving the overall effectiveness of the Career-Datacenter platform.
