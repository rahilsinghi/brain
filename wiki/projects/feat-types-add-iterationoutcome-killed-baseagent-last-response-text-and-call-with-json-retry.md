# feat(types): add IterationOutcome.KILLED, BaseAgent.last_response_text, and call_with_json_retry

This commit introduces foundational type and utility additions to the Ouroboros project in preparation for Phase 3 meta-learning capabilities. Key changes include a new KILLED outcome for safety invariant violations, cognitive trace telemetry via last_response_text, and a robust JSON parsing retry mechanism. Token usage tracking is also added to BaseAgent.

## Key Concepts

- **IterationOutcome.KILLED**: A new outcome state triggered when an agent violates safety invariants, enabling controlled termination of unsafe execution paths
- **BaseAgent.last_response_text**: A property that captures the raw text of an agent's last LLM response, enabling cognitive trace logging and telemetry
- **call_with_json_retry**: A utility function providing robust JSON parsing with automatic retry logic on empty or invalid responses
- **Token Accumulation**: New fields `total_input_tokens` and `total_output_tokens` on BaseAgent for tracking cumulative token usage across interactions
- **Phase 3 Meta-Learning**: The broader initiative this commit supports, focused on higher-order learning and self-improvement capabilities in the Ouroboros system

## Changes in Detail

### IterationOutcome.KILLED
A new enum value `KILLED` is added to the `IterationOutcome` type. This outcome is reserved for cases where an agent violates a defined safety invariant during execution. It provides a distinct, inspectable termination state separate from other outcomes such as success or error, enabling downstream logic to handle unsafe agent behavior explicitly.

### BaseAgent.last_response_text
A new attribute `last_response_text` is introduced on `BaseAgent`. This field stores the raw text of the most recent LLM response generated by the agent. Its primary purpose is to support telemetry pipelines and cognitive trace analysis, allowing observers and meta-learning systems to inspect what the agent "thought" during its last invocation.

### call_with_json_retry
A new utility function `call_with_json_retry` is added to provide resilient JSON parsing for LLM outputs. The function handles common failure modes including empty responses and malformed JSON by implementing a retry strategy, reducing brittleness in agent communication layers that depend on structured output.

### Token Accumulation on BaseAgent
Two new fields, `total_input_tokens` and `total_output_tokens`, are added to `BaseAgent`. These enable cumulative tracking of token consumption across an agent's lifetime, supporting cost monitoring, rate limiting, and meta-learning feedback loops that depend on resource usage signals.

## Context

This commit is explicitly framed as preparatory work for Phase 3 of the Ouroboros project. Phase 3 focuses on meta-learning, where agents may reason about and modify their own behavior. The KILLED outcome and telemetry features are safety and observability primitives needed before higher-order self-modification is enabled.
