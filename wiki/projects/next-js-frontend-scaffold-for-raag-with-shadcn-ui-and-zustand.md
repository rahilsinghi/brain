# Next.js Frontend Scaffold for raag with shadcn/ui and Zustand

This commit establishes the foundational Next.js frontend for the `raag` project, incorporating modern web development technologies. It sets up a robust development environment with TypeScript, Tailwind CSS, and shadcn/ui, alongside Zustand for state management and an API client for backend communication.

## Key Concepts

Next.js 16,TypeScript,Tailwind CSS 4,shadcn/ui,Zustand (State Management),API Client,Pydantic Models

## Details

This commit `2e6c13f` by Rahil Singhi, co-authored by Claude Opus 4.6, initializes the frontend scaffold for the `raag` project. The core technologies include:

*   **Next.js 16**: The React framework for server-side rendering and static site generation.
*   **TypeScript**: For type safety and improved developer experience.
*   **Tailwind CSS 4**: A utility-first CSS framework for rapid UI development.
*   **shadcn/ui**: A collection of reusable components built with Radix UI and Tailwind CSS.
*   **Zustand**: A small, fast, and scalable bear-necessities state management solution for React, specifically used here for chat state management.
*   **API Client**: Implemented to interact with the backend chat endpoint.
*   **TypeScript Interfaces**: Defined to ensure compatibility and type-safety with backend Pydantic models.

The `shadcn/ui` components integrated as part of this scaffold include: badge, button, card, input, scroll-area, sheet, avatar, separator, skeleton, textarea, and tooltip. This setup provides a solid, modern, and extensible base for further frontend development of the `raag` application.
