# Six Agent Tools for Destination Evaluation and Negotiation (Flock)

This commit introduces six specialized agent tools to the Flock repository, enabling AI-driven destination evaluation, hotel matching, and group travel negotiation. The tools combine deterministic constraint checking with Gemini-powered scoring and pgvector-based semantic similarity search. Together they form a pipeline for evaluating and compromising on travel destinations across multiple users.

## Key Concepts

- **Hard Constraint Checking**: Pure deterministic function (`check-hard-constraints.ts`) requiring no model call, ensuring baseline travel requirements are met before deeper evaluation
- **Gemini Destination Scoring**: `evaluate-destination.ts` uses Gemini to score destinations with human-readable reasons
- **Aesthetic Hotel Matching**: `get-aesthetic-match.ts` leverages pgvector's `match_hotels` RPC with 3072-dimensional Gemini Vision embeddings and cosine similarity
- **Compromise Generation**: `propose-compromise.ts` uses Gemini to synthesize group-acceptable destination proposals
- **Activity Search**: `search-activities.ts` queries a pre-seeded activities database for destination-relevant options
- **Per-User Trip Scoring**: `score-trip-decision.ts` produces individual match scores with a surprise factor for each user

## Tool Breakdown

### `check-hard-constraints.ts`
A pure function with no model invocation. Deterministically validates whether a destination satisfies non-negotiable user requirements (e.g., visa restrictions, budget ceilings, travel time limits) before any expensive model calls are made.

### `evaluate-destination.ts`
Calls the Gemini API to produce a scored evaluation of a destination, returning both a numeric score and structured reasoning. Enables explainable AI decisions in the destination selection flow.

### `get-aesthetic-match.ts`
Performs semantic hotel matching using pgvector's `match_hotels` RPC. Embeddings are 3072-dimensional vectors generated by Gemini Vision, compared via cosine similarity to find hotels that aesthetically align with user preferences.

### `propose-compromise.ts`
Uses Gemini to generate compromise destination proposals when group members have conflicting preferences. Operates on aggregated preference signals to find mutually acceptable options.

### `search-activities.ts`
Queries a pre-seeded activities database for a given destination, surfacing relevant experiences that can factor into scoring and negotiation.

### `score-trip-decision.ts`
Computes a per-user match score for a proposed trip decision, incorporating a surprise factor to account for serendipitous or unexpected but welcome destination attributes.

## Architecture Notes

- Hard constraints are evaluated deterministically to avoid unnecessary model inference costs.
- Vector similarity search operates at 3072 dimensions, consistent with Gemini Vision embedding output size.
- The toolset is designed to be composed within an agent loop, progressing from constraint filtering → scoring → matching → compromise generation.
