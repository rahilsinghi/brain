## What Maison Is
[[Maison]] is an AI-powered commercial platform designed for travel and hospitality. It deploys an [[AI Agent]] across various hotel touchpoints, including websites, mobile apps, SMS, WhatsApp, email, and internal tools. This AI handles guest interactions naturally, from answering questions and checking availability to fulfilling requests. Hotel operators manage the system via a dedicated console.

## Repo Access & Setup
*   **Monorepo:** The primary codebase is located in the `[[maison-agent]]` [[Monorepo]] on GitHub (`github.com/Maison-Labs/maison-agent`).
*   **Setup:** To get started, clone the repository, then run `make bootstrap` followed by `make dev`.
*   **Tech Stack:** The project is [[TypeScript]]-first, utilizing [[pnpm]] workspaces and [[Turborepo]] for monorepo management, and [[AWS CDK]] for infrastructure.
*   **Local URL:** Development is typically done at `https://local.maison-labs.com`.
*   **.env:** The `.env` file is generated by `make bootstrap` and requires a Firebase service account key file, stored outside the repository. Further details are available in the `#claude-onramp` Slack channel.
*   **Branch Convention:** Feature branches follow the `service/feature-name` format (e.g., `console/kb-analyzer`).
*   **Key Domains:** `maison.cx` (public site), `maison-labs.com` (platform).

## Architecture Overview
*   **API:** Built with [[Node.js]] and deployed on [[ECS Fargate]], supporting REST, A2A (JSON-RPC), and MCP interfaces.
*   **Database:** Primary data resides in [[PostgreSQL]] on Neon, with [[DynamoDB]] for sessions/tasks, [[Pinecone]] for vector search, and [[S3]] for file storage.
*   **Auth:** [[Firebase Auth]] is used for operators, while [[HMAC API Keys]] are employed for programmatic access.
*   **Console:** Uses [[A2UI]] schema-driven rendering, converting JSON schemas into HTML within a thin shell application.
*   **Real-time:** Chat message delivery leverages [[AppSync Events]] via [[WebSocket]]s.
*   **Observability:** Utilizes [[OpenTelemetry]] for distributed tracing, feeding into [[Grafana]] Tempo (traces), [[Loki]] (logs), and [[Langfuse]] for [[LLM]] tracing.
*   **Email:** Handled via [[SES]].
*   **Background:** Background processes are managed using [[SQS]] and [[Lambda]].

## Development Workflow
*   Initiate local development with `make bootstrap` then `make dev`.
*   All development is conducted on feature branches; direct commits to `main` are prohibited.
*   Fixes pushed to named branches are consolidated by [[Sandeep Baynes]].
*   The company mandates the use of [[Claude Code Vault Integration Documentation]] for all development tasks.
*   Issue tracking is managed via [[Linear (Issue Tracking)]], operating on an ongoing cycle without traditional sprint planning.
*   [[Conventional Commits]] are expected for all commits.
*   All infrastructure is managed as code using [[AWS CDK]] within the `infra/` directory.
