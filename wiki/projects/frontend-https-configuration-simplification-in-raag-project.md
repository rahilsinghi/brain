# Frontend HTTPS Configuration Simplification in Raag Project

This commit simplifies the frontend HTTPS configuration in the `rahilsinghi/raag` project. It removes the reliance on custom `mkcert` paths, opting instead for Next.js's built-in self-signed certificates. This change significantly reduces configuration friction for new project setups.

## Details

The commit `cb53520` in the `rahilsinghi/raag` repository, authored by Rahil Singhi on 2026-03-06, addresses a specific configuration challenge related to frontend HTTPS. Previously, the project's development environment for HTTPS relied on explicitly defined paths to certificates generated by `mkcert`. This approach introduced 'configuration friction,' particularly for developers setting up the project for the first time, as it necessitated additional manual steps to generate and specify these custom certificates.

To streamline the development workflow and reduce this friction, a change was implemented to leverage Next.js's inherent capability to generate and utilize its own self-signed certificates. By discontinuing the use of custom `mkcert` paths, the project's configuration becomes simpler and more self-contained. This modification benefits fresh clones of the repository by minimizing the manual setup steps required for HTTPS, thereby enhancing developer experience. This 'chore' commit involved a single file change with one addition and one deletion, indicating a precise and targeted adjustment to the project's development configuration.
