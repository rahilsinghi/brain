# Maison QA & Test Automation

This article details Maison's AI-driven QA system, which utilizes [[Claude]] and Playwright to create, run, and manage tests without requiring code. The system provides a web dashboard for test execution and results, aiming for a comprehensive test suite to be ready for staging to production deployment by late April.

## Key Concepts

AI-driven QA,Agentic testing,Playwright,Test suite,Self-healing tests,Intelligent test selection,Autonomous exploratory testing

## Details

Maison employs an advanced, agentic **AI-driven QA system** built upon Playwright. This system enables the creation and execution of tests through a web dashboard, with results stored alongside screenshots and screen recordings. A key feature is that **no code is required** to create or run these tests.

## Quality Dashboard URLs
The Quality Dashboard is accessible at various environments:
- Local: `local.maison-labs.com`
- Test: `quality.test.maison-labs.com`
- Staging: `quality.stg.maison-labs.com`
- Production: `quality.maison-labs.com`

Access requires credentials from the `.env` file: `QUALITY_ADMIN_EMAIL` and `QUALITY_ADMIN_PASSWORD`.

## Creating Tests
Tests are generated by describing them in plain language to [[Claude]]. For instance, a user might instruct:
> "Create a new quality UI test. Steps: open the login page at the Maison Labs login URL. Verify the Maison wordmark is visible. Verify the wordmark changes between light mode and dark mode."

**Tips for test creation:**
*   Be specific about the steps involved.
*   Include necessary credentials if the test requires them.
*   Maintain focus on a single test scenario.

Test suites are organized within the `apps/quality/src/suites/` directory, categorized by type (e.g., smoke, functional, visual, API, data).

## AI-Native Testing Strategy (Future)
Future enhancements to the testing strategy include:
1.  **Spec-to-test generation**: Automatic test creation from Linear ticket Acceptance Criteria.
2.  **Self-healing tests**: Automated repair of selectors or endpoints within tests.
3.  **Intelligent test selection**: AI analyzes code changes and selects the minimum necessary test set.
4.  **Autonomous exploratory testing**: AI navigates and explores the application without predefined scripts.
