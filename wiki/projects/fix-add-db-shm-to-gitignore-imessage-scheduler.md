# Fix: Add *.db-shm to .gitignore (iMessage Scheduler)

This commit resolves untracked file warnings in the `iMessage Scheduler` repository by adding `*.db-shm` to the `.gitignore` file. These files are SQLite WAL shared memory files that were causing Git status clutter after backend execution. The change ensures that these temporary database files are properly ignored.

## Details

The `iMessage Scheduler` project was experiencing persistent untracked file warnings after the backend process was run. This was due to `*.db-shm` files, which are shared memory files generated by SQLite's Write-Ahead Logging (WAL) mode, not being included in the `.gitignore` configuration. These files are temporary and do not need to be tracked by Git.

To address this, a fix was implemented in commit `b571b2c` by Rahil Singhi on 2026-03-10. This change involved modifying the project's `.gitignore` file to explicitly ignore all files matching the `*.db-shm` pattern. This prevents these temporary database artifacts from appearing as untracked files, cleaning up the Git status and development environment.
